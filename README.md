## 四、操作逻辑引擎（进攻型）

**尾盘买入**需同时满足：①午盘HOLD/BUY且下午未转弱 ②关联指数企稳 ③加仓后单基金≤30%且总仓≤90% ④无未知事件待落地
**尾盘卖出**满足任一：①单基金亏≥-8%强制止损 ②关联指数破位放量 ③板块单日暴涨>5%滞涨
**禁止买入**：大盘跳水(跌>0.5%) / 账户日亏>-3% / 未知事件待落地
**盘中监控触发**：单基金跌>5%(止损预警) / 上证破3460或20日线(减仓预警) / 单基金涨>15%(止盈预警)

## 五、预测命中率追踪

## 六、🌡️ 市场情绪因子分析（尾盘决策第四维）

尾盘决策从「技术+基本面+资金」三因子升级为**四因子**：新增「市场情绪面」逆向校准。

**8 大情绪因子 → 恐惧贪婪指数（0-100）**：

| 因子 | 权重 | 锚定 |
|------|------|------|
| 市场广度(涨跌家数比) | 15% | 涨/跌家数比 |
| 主力资金净流入 | 15% | 全市场/半导体主力净额 |
| VIX 恐慌指数 | 15% | 美股风险偏好 |
| 股债风险溢价 ERP | 15% | 权益相对债券便宜度 |
| 北向资金净买入 | 12% | 陆股通净买(亿) |
| 量能亢奋度 | 10% | 成交额 vs 20日均值 |
| 涨跌停家数 | 10% | 极端情绪 |
| 融资余额变化 | 8% | 杠杆追涨 |

**情绪分级与尾盘指令**：0-20极度恐惧(左侧买区) / 20-40恐惧 / 40-60中性(不干预) / 60-80贪婪 / 80-100极度贪婪(减仓预警)。

**铁律**：情绪信号**不覆盖** -8% 硬止损；仅影响加仓/止盈的弹性决策（恐惧区左侧分批加仓、贪婪区优先止盈）。

规范：`data/sentiment-protocol.md` ｜ 每日记录：`data/sentiment-log.json`（结构与 prediction-log 类似，可做情绪指标预测力回溯）。尾盘任务(2281411)已配置自动执行。

## 七、📈 板块日K量化分析（尾盘主力意图识别）

尾盘决策在「技术+基本面+资金+情绪」四因子基础上，新增**板块日K五维量化**——直接回答四个问题：主力是否给特殊信号？是否止跌/见顶？主力在洗盘建仓还是抢筹出货？收盘价有无特殊暗号？

**五维框架**（`data/kline-protocol.md`）：
| 维度 | 回答 | 输出 |
|------|------|------|
| 1. 趋势定位 | 多/空/震荡 | 趋势评分(价格vs MA5/10/20/60) |
| 2. 量价关系 | 量价配合/背离 | 量价健康度 |
| 3. 形态信号 | 止跌✓/见顶⚠ | 锤头/射击之星/吞没/背离 |
| 4. 主力行为 | 洗盘/建仓/抢筹/出货 | 依据口诀判定 |
| 5. 收盘价暗号 | 均线/光头光脚/缺口/地量天量 | 信号标签 |

**主力行为判别口诀**：不破支撑+缩量+尾盘拉回=洗盘(拿住) ｜ 低位小阳+温和放量=建仓(布局) ｜ 巨量长阳+突破=抢筹(跟进) ｜ 高位长上影+放量+净流出=出货(撤退)。

**多空倾向加权**：趋势40%+量价30%+形态15%+主力15% → ≥65偏多 / 35-64中性 / ≤34偏空。与情绪因子叠加（板块止跌+市场恐惧=高赔率左侧买点；板块见顶+市场贪婪=高确定止盈点）。每日信号写入 `data/kline-log.json`。尾盘任务(2281411)已配置自动执行。

## 八、📱 数据录入流程（重要）

### 支付宝净值更新时间
**支付宝准确净值每日 21:00 后更新**（T+1披露）。因此：
- 21:30 复盘任务使用当日准确T0数据
- 若需更早看盘，可用上一交易日净值做参考（标注"截至上一交易日"）

### 你如何提供数据
当前模型**不支持图片OCR**，请提供以下任一种**文本/数字**形式：

**方式A：支付宝持仓文本**（推荐，每日21:00后）
```
更新持仓：
东方人工智能 市值3000 +2.1% 持有收益+85
东方阿尔法科 市值2000 +2.5% 持有+30
永赢先锋半导体 市值2500 -3.8% 持有-70
...（每只一行）
```
→ 我解析并更新 `watchlist.json`，次日报告自动用新数据。

**方式B：养基宝场内穿透数据**（现在可发）
支付宝"养基宝"的基金组合**穿透到底层持仓**（股票/资产比例）。
提供后我可做：
- 验证基金实际行业暴露（是否真聚焦半导体/创新药）
- 检测风格漂移
- 增强调仓逻辑（穿透看底层，不止看净值）

**方式C：手动编辑**
直接改 `/workspace/daily-report/config/watchlist.json`（已含 strategy 字段）。

## 九、养基宝场内穿透数据说明

用户提供基金组合穿透持仓后，系统分析维度：
- **实际行业暴露**：基金底层股票的行业分布（半导体/医药/煤炭占比）
- **风格漂移检测**：名义主题 vs 实际持仓是否一致
- **集中度风险**：前十大持仓占比，是否过度集中
- **调仓匹配度**：底层资产与进攻型目标（半导体/创新药核心）的契合度

穿透数据让调仓从"看净值涨跌"升级为"看底层资产质量"。

## 十、📱 报告收件箱（移动端APP）

自动化任务生成的报告写到 `data/reports/YYYY-MM-DD/*.html`。为让你在手机/微信里像用 APP 一样查看，系统提供两层方案：

### 方案 A：bot 通道自动弹对话（需企业版+IM集成）
> ⚠️ 本沙箱会话未注入 bot 通道环境变量（CODEBUDDY_BOT_CHANNEL_* 为空），故任务为 personal 类型、只写文件不推对话。调度系统底层支持 `deliveryConfig.bot_channel` 推送——**若在企业微信/Slack 里的 CodeBuddy 机器人会话中运行任务，报告会直接发到对话**。要真正"自动弹微信"，需走该 bot 通道（详见对话记录）。

### 方案 B：GitHub Pages 托管（推荐，与登录方式无关）✅
把 `daily-report` 变成静态网站，生成一个 URL，微信/手机收藏即「基金报告APP」。**已为你搭好全部脚手架**：

- **`index.html`**（仓库根）：收件箱首页，移动端信息流，按日期分组，点开即看。
- **`publish.sh`**：重建收件箱 + `git add/commit/push`。**5 个自动化任务生成报告后自动调用它**，新报告秒级发布到站点。未配置 git 时优雅降级为仅本地更新。
- **`connect_repo.sh`**：一次性连接 GitHub 仓库（需你提供 PAT）。
- **`.nojekyll`**：禁用 Jekyll 处理。
- **`GITHUB_PAGES_SETUP.md`**：完整配置指南（建仓/开Pages/连仓库）。

**你的操作（仅一次性）**：
1. GitHub 新建仓库（如 `fund-reports`），Settings→Pages 选 main 分支 /root；
2. 生成 PAT（repo 写权限）；
3. 把「用户名 + 仓库名 + PAT」给我，我运行 `bash connect_repo.sh <用户> <仓库> <PAT>`；
4. 之后报告自动发布到 `https://<用户>.github.io/<仓库>/`，微信收藏即可。

> 安全：PAT 写入沙箱 git remote URL，可随时在 GitHub 撤销/轮换。

### 方案 C：云盘同步（连到电脑，推荐）✅
把报告自动同步到你电脑已登录的云盘（坚果云/OneDrive/阿里云盘/Dropbox），桌面客户端自动下载 → **报告直接连到电脑**。沙箱已装 `rclone v1.74.3`。

- **`sync_cloud.sh`**：用 rclone 把 `daily-report` 同步到 `.cloud_remote` 配置的远程（如 `nutstore:/daily-report`）。未配置则跳过。
- **`publish.sh`**：已整合云盘同步步骤，5 个任务生成报告后自动调用 → 自动同步到你电脑。
- **`CLOUD_SYNC_SETUP.md`**：各云盘配置指南。

**你的操作（一次性）**：
1. 在沙箱跑 `rclone config` 配好你的云盘远程（坚果云最省事：WebDAV + 邮箱 + 应用密码，无需 OAuth）；
2. 把远程名写进 `.cloud_remote`（如 `nutstore:/daily-report`）；
3. 你电脑装对应云盘客户端并登录同账号，把 `daily-report` 设为同步目录。

> 推荐**坚果云**：WebDAV 只需应用密码、无浏览器授权、Win/Mac 客户端自动同步，最适合"连到电脑"。
